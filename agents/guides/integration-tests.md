# Integration Tests

## Fixture Tests

- Fixtures live in `tests-integration/fixtures/{category}/{NNN_name}/`
- Use 3-digit numbered prefixes (e.g., `018_type_operator_valid`)
- Each fixture contains a `Main.purs` file
- Tests are auto-generated by `build.rs` and output to `tests/{category}/generated.rs`
- Snapshots go in `tests/{category}/snapshots/`

## Inline Tests

- For error-focused tests, use inline tests in `tests-integration/tests/{category}.rs`
- Use `empty_engine()` helper to create a test engine
- Assert on `checked.errors` with `insta::assert_debug_snapshot!`

## Snapshot Workflow

1. Run tests: `cargo nextest run -p tests-integration --test {category}`
2. Review new snapshots (`.snap.new` files)
3. Accept with `cargo insta accept`
